# Step 01 - 填寫專案資訊

> 初始化後，先填好 `openspec/project.md`，讓 AI 了解專案背景。
> 這份檔案會影響後續提案的品質，填越完整，AI 產出越貼近專案風格。

把以下提示貼給 AI，讓它讀取專案程式碼後自動填寫：

```
Please read openspec/project.md and help me fill it out
with details about my project, tech stack, and conventions
```

### project.md 通常包含

| 區塊 | 填寫內容 |
|------|----------|
| 專案描述 | 專案目的、主要功能 |
| 技術棧 | 語言、框架、資料庫、部署方式 |
| 撰寫慣例 | 命名規則、程式碼風格、目錄結構 |
| 重要限制 | 效能需求、安全性要求、相容性 |

### 既有專案的導入建議

如果是 Brownfield（既有系統），不需要一次補齊所有規格。
先從下一個要做的功能開始建立 spec，逐步累積即可。
