# Step 00 - 核心概念與安裝初始化

> Spec Kit 六步驟：`constitution` → `specify` → `clarify` → `plan` → `tasks` → `implement`
> 先定義規格再寫程式，讓 AI 產出可預測、可驗收的成果。

## 安裝

需先安裝 [uv](https://docs.astral.sh/uv/)，再執行：

```bash
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git
```

## 初始化

```bash
# 新專案
specify init my-project --ai claude

# 既有專案
specify init . --ai claude
```

系統會自動設定對應 AI 工具的 Slash Command。

## 初始化後的目錄

```
specs/
├── constitution.md   # 專案治理原則
├── spec.md           # 功能規格
├── clarifications.md # 釐清紀錄
├── plan.md           # 技術實作計畫
└── tasks.md          # 任務清單
```

## 常用指令

| Slash Command | 說明 |
|---------------|------|
| `/speckit.constitution` | 建立專案治理原則 |
| `/speckit.specify` | 描述要建造的功能（著重 what & why） |
| `/speckit.clarify` | 釐清規格中的模糊地帶 |
| `/speckit.plan` | 提供技術棧，產生實作計畫 |
| `/speckit.tasks` | 從計畫拆解出任務清單 |
| `/speckit.implement` | 按任務清單逐步實作 |
