學習如何在 Visual Studio Code 中使用 .instructions.md 和 .prompt.md 檔案來自訂 GitHub Copilot 回應，實現一致的專案特定 AI 協助

作者： Copilot That Jawn
中級程度

掌握指令和提示檔案的 Copilot 自訂設定

透過運用 Visual Studio Code 強大的自訂功能：指令檔案和提示檔案，將您的 GitHub Copilot 體驗提升到新層次。這些工具讓您能夠創建一致、專案特定的 AI 回應，理解您的編碼標準、偏好和常見工作流程。

參考資料 ：本指南基於官方 [Visual Studio Code Copilot 自訂設定文件](https://code.visualstudio.com/docs/copilot/copilot-customization)。如需最新的技術細節和 API 變更，請參閱官方文件。

了解自訂生態系統

Visual Studio Code 提供三種主要方式來自訂 Copilot 回應：

  1. .github/copilot-instructions.md - 全域專案指令
  2. .instructions.md 檔案 - 細部、情境特定指令
  3. .prompt.md 檔案 - 常見任務的可重複使用提示範本

設置指令檔案

全域專案指令

在專案根目錄建立 .github/copilot-instructions.md 檔案以設定專案層級的指導原則： 

---
# 範例：.github/copilot-instructions.md
---

# 專案編碼標準

## 一般指導原則
- 所有新的 JavaScript 程式碼使用 TypeScript
- 儘可能遵循函式式程式設計模式
- 偏好組合而非繼承
- 為所有公用 API 撰寫完整的 JSDoc 註解

## 測試要求
- 為所有業務邏輯包含單元測試
- 使用 Jest 作為測試框架
- 目標達成 80%+ 程式碼覆蓋率
- 為 API 端點撰寫整合測試

## 錯誤處理
- 在 React 元件中始終使用適當的錯誤邊界
- 使用結構化記錄來記錄錯誤
- 向使用者回傳有意義的錯誤訊息
- 使用適當的 try-catch 區塊處理非同步作業

啟用全域指令：

  1. 開啟 Visual Studio Code 設定（Ctrl+Shift+P → "Preferences: Open Settings"）
  2. 搜尋 github.copilot.chat.codeGeneration.useInstructionFiles
  3. 設定為 true

細部指令檔案

為特定場景建立目標明確的 .instructions.md 檔案：

了解指令檔案命名和前置資料

指令檔案必須遵循命名規範： [name].instructions.md

檔案命名範例：

  • react.instructions.md - React 元件指導原則
  • database.instructions.md - 資料庫開發模式
  • testing.instructions.md - 測試最佳實務
  • api-endpoints.instructions.md - API 開發標準
  • typescript.instructions.md - TypeScript 編碼規範
  • security.instructions.md - 安全審核指導原則

檔案位置選項：

  • 工作區 ： .github/instructions/ 資料夾（預設）
  • 使用者設定檔 ： %APPDATA%\Code\User\instructions\ （Windows）
  • 自訂位置 ：透過 chat.instructionsFilesLocations 設定進行配置

.instructions.md 檔案頂部的前置資料部分使用 YAML 語法，並支援數個重要屬性： 

---
applyTo: "glob-pattern"           # 必需：指定要套用指令的檔案
description: "簡短描述"            # 選用：描述指令的作用
---

重要前置資料屬性：

  • applyTo ：控制何時自動套用指令

    • "**" - 套用至工作區中的所有檔案
    • "**/*.ts" - 僅套用至 TypeScript 檔案
    • "**/*.{tsx,jsx}" - 套用至 React 元件檔案
    • "**/tests/**/*.ts" - 僅套用至測試檔案
    • "src/components/**/*" - 套用至特定目錄中的檔案

  • description ：在聊天視窗中滑鼠移至指令檔案上方時顯示的簡短說明

建議檔案名稱的完整檔案範例：

檔案： react.instructions.md 

---
applyTo: "**/*.{tsx,jsx}"
description: "React 元件開發指導原則"
---

檔案： database.instructions.md 

---
applyTo: "**/models/**/*.ts"
description: "Entity Framework 模式和資料庫指導原則"
---

檔案： testing.instructions.md 

---
applyTo: "**/tests/**/*.{ts,js}"
description: "測試最佳實務和模式"
---

檔案： api-endpoints.instructions.md 

---
applyTo: "src/api/**/*.ts"
description: "RESTful API 端點開發指導原則"
---

框架特定指令

範例檔案： react.instructions.md 

---
applyTo: "**/*.{tsx,jsx}"
description: "React 元件開發指導原則"
---

# React 元件指令

## 元件結構
- 使用帶有 hooks 的函式元件
- 使用 TypeScript 介面實作適當的 prop 驗證
- 提取自訂 hooks 以供重複使用邏輯
- 需要時使用 React.memo 進行效能最佳化

## 狀態管理
- 使用 useState 管理本地元件狀態
- 使用 useReducer 處理複雜狀態邏輯
- 實作自訂 hooks 以共享狀態邏輯
- 避免 prop drilling - 適時使用 Context API

## 樣式設計
- 使用 CSS Modules 或 styled-components
- 遵循 BEM 方法論進行類別命名
- 實作行動優先的響應式設計
- 使用 CSS 自訂屬性進行主題設計

資料庫特定指令

範例檔案： database.instructions.md 

---
applyTo: "**/models/**/*.ts"
description: "資料庫模型和查詢指導原則"
---

# 資料庫開發指導原則

## Entity Framework 模式
- 使用儲存庫模式進行資料存取
- 實作適當的實體關係
- 包含稽核欄位（CreatedAt、UpdatedAt、CreatedBy）
- 對複雜屬性使用值物件

## 查詢最佳化
- 始終使用參數化查詢
- 實作適當的索引策略
- 對所有資料庫作業使用非同步方法
- 包含適當的資料庫例外錯誤處理

透過 Visual Studio Code 建立指令檔案

快速建立方法：

  1. 在 Visual Studio Code 中開啟聊天視窗
  2. 點選「配置聊天」按鈕（齒輪圖示）
  3. 選擇「指令」→「新增指令檔案」
  4. 選擇工作區或使用者設定檔位置
  5. 為檔案命名並開始撰寫指令

命令調色盤方法：

  • 按 Ctrl+Shift+P
  • 輸入「Chat: New Instructions File」
  • 依照提示操作

掌握提示檔案

提示檔案（ .prompt.md ）是常見開發任務的可重複使用範本。

了解提示檔案命名和前置資料

提示檔案必須遵循命名規範： [name].prompt.md

檔案命名範例：

  • create-react-component.prompt.md - 產生 React 元件
  • security-review.prompt.md - 執行安全程式碼審核
  • api-endpoint.prompt.md - 建立 RESTful API 端點
  • generate-tests.prompt.md - 建立單元和整合測試
  • refactor-code.prompt.md - 程式碼重構工作流程
  • generate-docs.prompt.md - 文件產生

檔案位置選項：

  • 工作區 ： .github/prompts/ 資料夾（預設）
  • 使用者設定檔 ： %APPDATA%\Code\User\prompts\ （Windows）
  • 自訂位置 ：透過 chat.promptFilesLocations 設定進行配置

提示檔案使用 YAML 前置資料來配置提示的執行方式：

---
mode: "ask"|"edit"|"agent"        # 選用：使用的聊天模式（預設："agent"）
model: "gpt-4"|"gpt-3.5-turbo"    # 選用：要使用的特定 AI 模型
tools: ["tool1", "tool2"]         # 選用：代理模式中可用的工具
description: "簡短描述"           # 選用：提示的作用說明
---

前置資料屬性說明：

  • mode ：決定提示的運作方式

    • "ask" - 簡單的問答模式
    • "edit" - 直接檔案編輯模式
    • "agent" - 具有工具存取的完整代理模式（預設）

  • model ：指定要使用的 AI 模型

    • "gpt-4" - 最強大的模型，適用於複雜任務
    • "gpt-3.5-turbo" - 更快速、更經濟，適用於較簡單的任務
    • 如果未指定，則使用 Visual Studio Code 中目前選擇的模型

  • tools ：在代理模式中可用的工具名稱/組合陣列

    • ["typescript"] - TypeScript 特定工具
    • ["react", "typescript"] - 多個工具組合
    • ["web-search", "file-system"] - 外部工具
    • 透過聊天中的「配置工具」來配置可用工具

  • description ：提示用途的簡短說明（在 UI 中顯示）

常用前置資料模式：

# 用於程式碼產生任務
---
mode: "agent"
model: "gpt-4"
tools: ["typescript", "react"]
description: "產生包含測試的 React 元件"
---

# 用於程式碼審核任務
---
mode: "ask"
description: "執行安全程式碼審核"
---

# 用於檔案編輯任務
---
mode: "edit"
model: "gpt-4"
description: "按照團隊標準重構程式碼"
---

# 用於文件任務
---
mode: "agent"
tools: ["markdown", "web-search"]
description: "產生完整的 API 文件"
---

基本提示檔案結構

範例檔案： create-react-component.prompt.md 

---
mode: "agent"
model: "gpt-4"
tools: ["typescript", "react"]
description: "使用 TypeScript 產生 React 元件"
---

# 建立 React 元件

根據以下要求建立新的 React 元件：

## 元件規格
- 元件名稱：${input:componentName:輸入元件名稱}
- 使用 TypeScript 搭配適當的介面
- 包含適當的 prop 驗證
- 實作響應式設計
- 新增完整的 JSDoc 註解

## 額外上下文
- 遵循我們專案的編碼標準：[coding-standards](.github/copilot-instructions.md)
- 參考現有元件：#src/components/
- 使用我們的設計系統：#src/styles/design-system.ts

## 輸出要求
- 產生元件檔案
- 建立對應的測試檔案
- 在註解中包含使用範例
- 適當地匯出元件

選擇的上下文檔案：${selection}

Advanced Prompt Examples

Security Review Prompt

Example File: security-review.prompt.md 

---
mode: "agent"
description: "Perform comprehensive security review"
---

# Security Code Review

Perform a thorough security analysis of the selected code:

## Review Areas
1. **Input Validation**: Check for proper sanitization and validation
2. **Authentication**: Verify proper auth implementation
3. **Authorization**: Ensure proper access controls
4. **Data Exposure**: Look for sensitive data leaks
5. **SQL Injection**: Check for parameterized queries
6. **XSS Prevention**: Verify proper output encoding

## Context Files
- Security guidelines: [security-guidelines](./docs/security.md)
- Authentication service: #src/services/auth.ts

## Output Format
Provide findings in this format:
- **Issue**: Description of the security concern
- **Severity**: Critical/High/Medium/Low
- **Location**: File and line number
- **Recommendation**: Specific fix instructions
- **Example**: Code example showing the fix

Selected code: ${selection}

API Generation Prompt

Example File: api-endpoint.prompt.md 

---
mode: "edit"
description: "Generate RESTful API endpoint"
---

# Generate API Endpoint

Create a complete RESTful API endpoint with the following specifications:

## Requirements
- HTTP method: ${input:method:GET|POST|PUT|DELETE}
- Resource: ${input:resource:Enter resource name}
- Include proper validation middleware
- Implement error handling
- Add comprehensive OpenAPI documentation
- Include unit and integration tests

## Dependencies
- Follow API patterns: [api-patterns](./docs/api-patterns.md)
- Use validation schemas: #src/schemas/
- Reference existing endpoints: #src/controllers/

## File Structure
Create these files:
- Controller: `src/controllers/${input:resource}Controller.ts`
- Routes: `src/routes/${input:resource}Routes.ts`
- Tests: `tests/controllers/${input:resource}Controller.test.ts`
- Schema: `src/schemas/${input:resource}Schema.ts`

Target location: ${workspaceFolder}/src

Using Prompt Files

Running Prompts

Via Chat Input:

  • Type / followed by prompt name: /create-react-component
  • Pass parameters: /create-api-endpoint: method=POST resource=users

Via Command Palette:

  • Press Ctrl+Shift+P
  • Type "Chat: Run Prompt"
  • Select your prompt file

Via Editor:

  • Open the .prompt.md file
  • Click the play button in the editor title
  • Choose to run in current or new chat session

Advanced Customization Techniques

Variable System

Leverage Visual Studio Code's variable system in your instructions and prompts:

# Current workspace: ${workspaceFolder}
# Current file: ${file}
# Selected text: ${selectedText}
# User input: ${input:variableName:placeholder}

File References

Link to other files and instructions:

# Reference other instructions
Follow guidelines in [typescript-rules](./typescript.instructions.md)

# Reference workspace files
Check existing patterns: #src/utils/helpers.ts

# Reference with markdown links
See [API documentation](../docs/api.md) for details

Conditional Instructions

Create context-aware instructions using the applyTo property: 

---
applyTo: "**/tests/**/*.ts"
description: "Testing-specific instructions"
---

# Testing Guidelines
- Use describe/it pattern for test structure
- Include setup and teardown as needed
- Mock external dependencies
- Test both success and error scenarios

    ├── create-react-component.prompt.md    # Generate React components

User Profile Instructions

Store personal preferences in your user profile:

  • Location : %APPDATA%\Code\User\prompts\ (Windows)
  • Sync : Enable Settings Sync to share across devices
  • Access : Available in all workspaces

Team Collaboration Best Practices

Version Control

  • Commit instruction and prompt files to your repository
  • Document your customization strategy in README
  • Review changes to instructions like code changes
  • Use descriptive commit messages for instruction updates

Team Standards

---
# Example: .github/copilot-instructions.md for team
---

# Team Development Standards

## Code Review Requirements
- All PRs require 2 approvals
- Include tests for new features
- Update documentation for API changes
- Follow conventional commit format

## Architecture Decisions
- Use clean architecture patterns
- Implement dependency injection
- Follow SOLID principles
- Document architectural decisions in ADRs

Knowledge Sharing

  • Create prompt libraries for common team tasks
  • Share useful instructions via team documentation
  • Conduct training sessions on customization techniques
  • Maintain a team wiki of best practices

Troubleshooting Common Issues

Instructions Not Applied

  1. Check Settings : Ensure github.copilot.chat.codeGeneration.useInstructionFiles is enabled
  2. File Location : Verify files are in correct directories
  3. Syntax : Check YAML frontmatter formatting
  4. File Extension : Ensure .instructions.md or .prompt.md extensions

Conflicting Instructions

  • Avoid contradictory guidelines across files
  • Use specific applyTo patterns to prevent conflicts
  • Test instruction combinations before deployment
  • Document instruction hierarchy and precedence

Performance Considerations

  • Keep instruction files concise and focused
  • Avoid overly complex nested references
  • Monitor token usage in complex setups
  • Regular review and cleanup of unused instructions

Measuring Effectiveness

Success Metrics

  • Consistency : Are generated responses following your guidelines?
  • Efficiency : Are common tasks faster with prompts?
  • Quality : Is code quality improving with instructions?
  • Adoption : Is the team using the customizations?

Iterative Improvement

  1. Start with basic instructions
  2. Gather feedback from team usage
  3. Refine and expand based on common patterns
  4. Document lessons learned
  5. Share successful patterns with the team

結論

指令檔案和提示檔案將 GitHub Copilot 從通用助理轉變為專案感知、團隊一致的開發夥伴。從基本指令開始，逐步建立您的提示庫，並根據團隊需求持續改善。

成功的關鍵在於一致性、團隊採用和反覆改進。從工作流程中最常見的任務開始，然後隨著您發現自動化和標準化的新機會，擴展您的自訂庫。

下一步：

  1. 建立您的第一個 .github/copilot-instructions.md 檔案

  4. 與您的團隊分享和反覆運算

請記住：這些自訂設定不僅僅是為了讓 Copilot 運作得更好，更是將您團隊的知識和最佳實務編纂成可重複使用、可擴展的工具，幫助在整個程式碼庫中維護一致性和品質。
