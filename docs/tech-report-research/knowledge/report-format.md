# 報告格式與完成檢查

## 觸發時機

- 開始撰寫 report.md 時
- 調整段落結構與 Markdown 格式時
- 交付前檢查 TODO 與內容完整性時

## 內容

### 固定段落

技術報告應固定為四段：

1. 總結
2. 亮點
3. 趨勢分析
4. 參考資料

所有內容都需標記來源編號；若使用非來源所蒐集資料，需特別提醒。

### Markdown 格式

- 段落使用 H2
- 分類使用 H3
- 重要名詞與專有詞彙使用粗體
- 參考資料使用 Markdown 表格
- 用詞須使用繁體中文與台灣用語
- 中英文之間保留適當空格

### 完成前檢查

- 至少檢查 TODO 文件一次
- 補上 TODO 中尚未完成的內容
- 確認是否有缺少領域內資料需要補足
- 確認資訊權重是否需要調整
- 確認搜尋時間範圍是否正確
