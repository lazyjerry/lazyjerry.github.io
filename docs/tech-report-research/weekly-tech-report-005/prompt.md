# 任務請求：科技服務資訊週記

## 今天日期

2026-05-30

## 報告標題格式

`2026-05-30 科技服務資訊週記`

## Meta 格式

`作者：Claude Code　報告日期：2026-05-30`

- 目前使用的 AI 工具：Claude Code（Claude Opus 4.8）

## 讀者目標

以程式設計師為主要讀者，內容除軟體工程外，也涵蓋科技應用與科技服務相關資訊。

## 必要分類

- AI 科技
- 軟體工程
- 軟硬體資訊安全
- 硬體或軟硬整合
- 網路/伺服器等基礎
- 金融科技
- 醫療科技
- 運輸物流
- 房地產與室內外裝潢
- 現場表演藝術
- 影視音樂
- 其他領域科技應用突破

若出現無法歸類的重要領域，可動態補充分類。

## 四大段落要求

1. **總結**：100 至 300 字，依實際內容統整重點，標記來源編號。
2. **亮點**：依分類條列，每條先一句話下結論，再用約 100 至 300 字說明，儘量加入具體例子或譬喻，標記來源編號。
3. **趨勢分析**：依分類整理，歸納可能趨勢或更深層觀察，儘量加入具體例子或譬喻，標記來源編號。
4. **參考資料**：依分類與時間列出來源，欄位至少含：來源編號、文章標題與 Google 搜尋連結、一句話繁體中文結論或亮點、發布日期、來源網址與名稱、分類。

## 來源時間範圍與權重規則

- 僅使用 2026-05-15 至 2026-05-30（今天往前 15 天）內發布的資料。
- 優先使用有明確日期標記的來源。
- 無法確認是否落在 15 天內者，直接排除。
- 確認在 15 天內但無精確日期者，可標記 `*未知時間`。
- 中國來源、社群平台（知乎、Reddit 等）優先以高權重資料替換，無法替換則刪除。
- 台灣主流媒體與權威科技來源優先。
- 權重分數以 `1-10/10` 標記。

## source.md 蒐集格式要求

每筆至少包含：來源網址、繁體中文標題、100 至 500 字繁體中文摘要、權重分數（1-10/10）、發布日期、分類、是否納入正式參考資料。淘汰來源需註明原因。

## 引用格式

- 文中引用使用 `[[{數字}. {資料來源名稱}]](#ref-{數字})`。
- 參考資料以表格呈現，並在每列前加 `<a id="ref-{數字}"></a>`。
- 內文引用編號與參考資料表編號需一致。

## 完成前檢查要求

- 各分類是否缺少資料
- 資訊權重是否足夠
- 搜尋時間範圍是否正確（2026-05-15 至 2026-05-30）
- 資料來源是否正確（網址、來源網站名稱、發布日期、引用對應）
- 文章內引用的來源編號與參考資料表編號是否一致
- 錯誤來源或整理內容是否已修正
