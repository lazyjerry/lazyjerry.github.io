## 總結

本期最清楚的線索是 **AI 代理從展示走向營運化**。Google、xAI、OpenAI 都把代理能力往產品深處放：模型、程式開發、資安工作流各走一條路。企業端的壓力也跟著變得很實際，包含伺服器、液冷、高速交換機與用電 [[1. 中央社]](#ref-1) [[2. iThome]](#ref-2) [[3. iThome]](#ref-3) [[7. 中央社]](#ref-7) [[8. 中央社]](#ref-8)。金融、醫療、物流、PropTech、沉浸式內容、音訊與農漁業案例則把重點拉回執行面：AI 要有價值，得接進資料、場域、法規與人機協作流程 [[9. iThome]](#ref-9) [[10. iThome]](#ref-10) [[12. 中央社]](#ref-12) [[15. 中央社訊息平台]](#ref-15)。

<a id="highlights"></a>
## 亮點

<a id="hl-ai"></a>
### AI 科技
- **模型廠商把 AI 代理放到產品前排。** Google I/O 2026 發表 Gemini 3.5 系列，3.5 Flash 主打代理任務與行動能力，也開放開發者透過 Google Antigravity、Google AI Studio、Android Studio 的 Gemini API 使用。模型競爭正在從「回答得好」往「能不能完成跨工具、多步驟、長時間任務」移動 [[1. 中央社]](#ref-1)。

<a id="hl-software"></a>
### 軟體工程
- **程式代理工具開始走進 CLI 與多代理協作。** xAI 的 Grok Build 以命令列介面切入專業開發者的日常工作，可修改程式、更新文件、排查效能問題，並提供 Plan Mode 與子代理平行處理。工程團隊接下來要看的除了模型能力，還有計畫審查、變更透明度、專案規範整合與人類核准流程 [[2. iThome]](#ref-2)。

<a id="hl-security"></a>
### 軟硬體資訊安全
- **資安防禦被推進日常開發循環。** OpenAI Daybreak 以 GPT-5.5-Cyber 與 Codex Security 支援安全程式碼審查、威脅模型、漏洞分級、惡意程式分析與修補驗證；資安署也提醒企業把漏洞管理視為營運風險，預防、偵測、應變與復原都要顧到 [[3. iThome]](#ref-3) [[4. 中央社]](#ref-4)。

<a id="hl-hardware"></a>
### 硬體或軟硬整合
- **AI 伺服器競爭已經延伸到 GPU 之外。** TechNews 引述通路訪查指出，4 月全球 GB200 與 GB300 AI 機櫃產量約 8,300 櫃，鴻海以約 3,700 櫃、逾 44% 市占領先。短期波動來自 GB200 轉 GB300、液冷組件調整與供貨節奏，但需求面仍偏強 [[6. TechNews]](#ref-6)。

<a id="hl-infrastructure"></a>
### 網路/伺服器等基礎
- **AI 基礎設施壓力同步打到網路、能源與供應鏈。** 鴻海表示 AI 機櫃出貨倍增，800G 以上交換機全年成長倍增，CPO 交換機預計第 3 季量產；DIGITIMES 也估代理式 AI 將推升 AI 與通用伺服器出貨。資料中心用電壓力，讓現場能源解方被拉到檯面上 [[7. 中央社]](#ref-7) [[8. 中央社]](#ref-8)。

<a id="hl-fintech"></a>
### 金融科技
- **AI 開始碰到個人金融資料，也推高監理與詐騙壓力。** OpenAI 在美預覽 ChatGPT 連結個人金融帳戶，透過 Plaid 串接銀行與支付 App；同週 BaFin 與 Visa 相關報告則提示金融業要面對更快的弱點利用、AI 社交工程與數位營運韌性要求 [[10. iThome]](#ref-10) [[5. iThome]](#ref-5) [[11. 中央社訊息平台]](#ref-11)。

<a id="hl-healthtech"></a>
### 醫療科技
- **醫療 AI 的關鍵開始轉向資料標準與制度導入。** iThome MedTech 雙周報提到美國醫院協會啟動加速器，協助醫院導入 AI、虛擬照護與 EHR；臺灣健保署與 CDE 持續與英國 NICE 合作，聚焦 HTA、癌症數位治理與 FHIR 標準化 [[9. iThome]](#ref-9)。

<a id="hl-logistics"></a>
### 運輸物流
- **無人載具與自駕走向驗證場域與量產。** XPONENTIAL Detroit 2026 台灣館規模創歷年最大，台灣也與美國單位簽署合作備忘錄，聚焦自主飛行、城市交通與測試場域；小鵬則啟動 robotaxi 量產，目標 Level 4 自動駕駛，但政策與試營運仍是主要風險 [[12. 中央社]](#ref-12) [[13. TechNews]](#ref-13)。

<a id="hl-proptech"></a>
### 房地產與室內外裝潢
- **PropTech 開始進入商用不動產營運流程。** ICSC+PROPTECH 2026 聚焦 AI 驅動租賃與商業工具、智慧建築、物業管理軟體與租戶互動平台。房地產科技的重點已經延伸到招商、維運、物業管理與決策流程 [[14. ICSC]](#ref-14)。

<a id="hl-arts"></a>
### 現場表演藝術
- **沉浸式內容成為臺灣藝術科技出海管道。** 《與火共舞：王羽佳鋼琴藝術沉浸展》入圍坎城影展沉浸式競賽單元，結合 XR 與實景娛樂，以多感官方式重新詮釋古典音樂。這類作品已把舞台延伸到影像、空間運算、觀眾動線與國際發行 [[15. 中央社訊息平台]](#ref-15)。

<a id="hl-entertainment"></a>
### 影視音樂
- **AI 音訊生成開始變成個人化資訊入口。** Spotify Studio 可連接使用者授權的信箱、行事曆、筆記、文件、新聞網站與音樂目錄，再用自然語言生成聲音簡報、新聞摘要與行程整理。音訊平台的競爭正在從串流內容，擴展到個人資料整理與 AI 生成節目 [[16. iThome]](#ref-16)。

<a id="hl-others"></a>
### 其他領域科技應用突破
- **AI 的實用價值出現在農漁業與無障礙場景。** 國合會帶 22 國農業官員訪問海大，了解智慧養殖、精準水產管理與 AI 生產決策；Apple 也預告 Apple Intelligence 輔助使用功能，包含影像描述、自然語言語音控制與裝置端字幕生成 [[17. 中央社訊息平台]](#ref-17) [[18. 中央社]](#ref-18)。

<a id="trends"></a>
## 趨勢分析

<a id="tr-ai"></a>
### AI 科技
AI 模型競爭正從單次互動轉向「可持續執行任務」。Gemini 3.5 把速度、成本、代理任務、API 與開發平台綁在一起，目的很明確：模型供應商想把角色從文字輸出工具推進到工作流執行層 [[1. 中央社]](#ref-1)。**未來想像：作業系統（OS）與 AI 模型的邊界將會完全模糊化。AI 將不再是我們被動呼叫的「工具」，而是常駐於背景主動協調軟硬體資源的「虛擬數位同事」。這種轉變將催生出全新的「代理人間協作（Agent-to-Agent）」生態系，人類的指令只需下達一次，後續的多步驟協調皆由 AI 網路自主完成。**

<a id="tr-software"></a>
### 軟體工程
AI coding 的採用風險會從「產生錯誤程式」擴大到「錯誤變更被代理自動擴散」。Plan Mode、子代理與變更清單會變成基本安全欄杆；工程團隊也得把代理輸出接回 code review、測試、權限控管與稽核紀錄 [[2. iThome]](#ref-2) [[3. iThome]](#ref-3)。**未來想像：軟體工程師的角色將從「程式碼撰寫者」轉變為「AI 開發者艦隊指揮官」。未來的軟體開發不再是逐行編寫，而是專注於系統架構設計、提示詞工程（Prompt Engineering）以及設定嚴格的邏輯與道德邊界，繁瑣的程式碼生成、重構與最佳化將全權由 AI 代理自治完成。**

<a id="tr-security"></a>
### 軟硬體資訊安全
AI 讓攻防兩邊都加速。防禦方能用 Daybreak 類工具做威脅模型與修補驗證，攻擊方也可能更快串接漏洞。資安署把重心放在韌性、備份、MFA、Passkey 與縮短偵測反應時間，這很務實；未來資安能力會更常用恢復時間與損害控制來衡量 [[3. iThome]](#ref-3) [[4. 中央社]](#ref-4)。**未來想像：「AI 對抗 AI」的自動化網路戰將成為常態。防禦方將部署全天候自主巡邏的 AI 代理，在威脅發生前進行即時弱點預測與自我修復。傳統的零信任架構（Zero Trust）將進化為「AI 演算法持續驗證（AI-Verified Trust）」，連線的合法性將基於行為模式的毫秒級判定。**

<a id="tr-hardware"></a>
### 硬體或軟硬整合
AI 伺服器已經進入整櫃、液冷、電源、高速傳輸與系統驗證一起競爭的階段。GB200 轉 GB300 的短期供應波動，也提醒供應鏈一件事：每一代平台切換，都會牽動散熱、快速接頭、供電與組裝節奏 [[6. TechNews]](#ref-6) [[7. 中央社]](#ref-7)。**未來想像：未來的硬體設計將逐漸由 AI 進行「協同生成式設計」，打破人類工程師的思考框架。面對能耗牆，未來的伺服器架構可能朝向「仿生學」或極高密度的單一超級運算大腦發展，甚至加速矽光子以外的量子混合晶片研發，以追求極致的「每瓦智慧力（Intelligence per Watt）」。**

<a id="tr-infrastructure"></a>
### 網路/伺服器等基礎
代理式 AI 的需求會把資料中心推向更高密度，也要求更高可用性。CPO 交換機、800G 以上網路、表後電力與 SOFC 都會直接影響 AI 服務能不能穩定供應。未來做工程決策時，模型、網路、電力與碳成本會被一起放上桌 [[7. 中央社]](#ref-7) [[8. 中央社]](#ref-8)。**未來想像：資料中心將去中心化，演變成「AI 能源運算微電網」。為了達到能源自主與極致冷卻，未來的基礎設施可能會大規模移往極端環境（如深海、極地甚至太空邊緣）。運算力將像水和電一樣，成為隨需取用、能跨國無縫調度的無形基礎設施。**

<a id="tr-fintech"></a>
### 金融科技
AI 金融助理一旦接上銀行帳戶，便利性和風險會一起放大。使用者會期待 AI 協助預算、投資與財務規劃；監理機關則會要求金融機構更快修補弱點、證明營運韌性，並降低 AI 社交工程詐騙造成的信任破口 [[10. iThome]](#ref-10) [[5. iThome]](#ref-5) [[11. 中央社訊息平台]](#ref-11)。**未來想像：「演算法身分（Algorithmic Identity）」將成為新的資產階級，每個人都將擁有超個人化的自主財富管理代理。更深遠的影響是，背景中可能出現大量的「AI 對 AI 微型交易」，在人類無感的情況下即時最佳化資產配置與日常消費，徹底顛覆傳統貨幣流動與商業轉換的樣貌。**

<a id="tr-healthtech"></a>
### 醫療科技
醫療科技的瓶頸常落在制度與資料，模型展示通常比較容易。FHIR、HTA、癌症數位治理與醫院加速器都指向同一個條件：醫療 AI 必須進入 EHR、支付評估、真實世界資料與臨床決策流程，才有機會變成可持續的服務 [[9. iThome]](#ref-9)。**未來想像：醫療產業將從「生病後的治療」徹底轉向「預測性的持續健康最佳化」。AI 將成為每位病患終身的「數位孿生（Digital Twin）」，在進行物理治療前，所有的藥物交互作用與手術過程都會先在數位分身上進行沙盤推演，實現百分之百的精準醫療。**

<a id="tr-logistics"></a>
### 運輸物流
無人機與 robotaxi 的關鍵正在從演算法轉向驗證場域、法規、冗餘與供應鏈。台灣無人機展出重點放在美國測試場域與關鍵模組，小鵬 robotaxi 則面對 L4 許可與商轉節奏的不確定性。運輸科技接下來跑多快，政策與安全工程會比宣傳影片更有影響力 [[12. 中央社]](#ref-12) [[13. TechNews]](#ref-13)。**未來想像：未來的城市物流將是一場「無聲的群蜂之舞」。蜂群智慧（Swarm Intelligence）將主導無人機與自駕車隊，把整座城市變成動態的高效路由網路。私人載具的擁有權將大幅下降，取而代之的是「物流與移動即服務（MaaS/LaaS）」的全天候隱形基礎設施。**

<a id="tr-proptech"></a>
### 房地產與室內外裝潢
PropTech 的 AI 應用會先在高頻、可量化、可追蹤的商用流程產生價值，例如租賃、物業管理、租戶互動與智慧建築維運。室內外裝潢與空間規劃也會慢慢吃到資料化資產、視覺化方案與營運回饋的好處 [[14. ICSC]](#ref-14)。**未來想像：未來的建築物將變成具備「感知能力」的實體。它們不再是冰冷的水泥空間，而是能根據內部人員的情緒、生理狀態與外部氣候，動態調整光線、溫溼度甚至室內隔間的生命體；生成式空間設計將賦予實體環境隨時間自我進化的能力。**

<a id="tr-arts"></a>
### 現場表演藝術
XR 與 LBE 讓現場表演有機會同時經營場館票房與國際內容授權。古典音樂、沉浸式影像與實景娛樂結合後，創作者要處理的事情變多了：藝術敘事、互動設計、空間技術與跨國發行都躲不掉 [[15. 中央社訊息平台]](#ref-15)。**未來想像：「固定敘事」的表演形式將逐漸消退，未來的藝術表演將是充滿變數的活體生態系。AI 系統會即時讀取現場觀眾的生物特徵（如心跳、視線變化）與情緒回饋，當下動態改變劇情走向、光影與音樂節奏。觀眾、表演者與 AI 將在當下共同成為藝術的創作者。**

<a id="tr-entertainment"></a>
### 影視音樂
AI 音訊工具會讓 Podcast 與音樂平台變成「資料轉聲音」的介面。Spotify Studio 類產品若被廣泛採用，創作者與平台之間的邊界會改變：部分內容會由使用者資料即時生成，版權、資料授權與個人化推薦也會綁得更緊 [[16. iThome]](#ref-16) [[15. 中央社訊息平台]](#ref-15)。**未來想像：這暗示著傳統大眾單向傳播媒體的式微，迎接「提示詞隨選串流（Prompt-based Streaming）」的時代。影音內容將依據觀看者當下的心理狀態瞬間生成，只需給出一個情境或心情，AI 就能在幾秒鐘內為您渲染出一整部專屬電影或一小時的音樂專輯。**

<a id="tr-others"></a>
### 其他領域科技應用突破
智慧養殖與 AI 輔助使用顯示，痛點明確、容錯低的日常問題，反而更容易看出 AI 的價值。農漁業要面對氣候與人力壓力，無障礙功能則要降低資訊取得與操作門檻；兩者都要求 AI 與感測、裝置端運算、隱私與現場流程整合 [[17. 中央社訊息平台]](#ref-17) [[18. 中央社]](#ref-18)。**未來想像：AI 在特定領域最大的貢獻將是「能力的極致平權」。在農業，這意味著行星級別的氣候調適能直接落實到微型農場的日常決策；在無障礙領域，這代表著介面屏障的徹底消除，意念或極微小的動作都能無縫轉化為數位指令，讓科技真正成為人類身體的延伸。**
