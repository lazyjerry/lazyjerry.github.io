### 2. GitHub Copilot Agent Mode 與 MCP
- **日期**: 2025年12月30日
- **來源**: GitHub Blog
- **URL**: https://github.blog/developer-skills/agentic-ai-mcp-and-spec-driven-development-top-blog-posts-of-2025/
- **關鍵內容**:
  - Agent Mode（2月發布）：可以迭代自己的代碼、識別錯誤並實時修復
  - Coding Agent（5月發布）：嵌入 GitHub，可探索存儲庫、編寫代碼、通過測試並開啟 PR
  - Agent HQ（10月 GitHub Universe 發布）：統一所有 AI 代理的生態系統平台
  - MCP（Model Context Protocol）：讓 AI 代理和工具互相溝通
  - 4月推出 Agent Mode with MCP support
  - 9月推出 GitHub MCP Registry
  - Spec-driven development：先寫規範再編碼的開發方式
  - Spec Kit：開源工具包，將規範驅動開發帶入編碼代理工作流程

### 3. React2Shell (CVE-2025-55182) 漏洞
- **日期**: 2025年12月15日
- **來源**: Microsoft Security Blog
- **URL**: https://www.microsoft.com/en-us/security/blog/2025/12/15/defending-against-the-cve-2025-55182-react2shell-vulnerability-in-react-server-components/
- **關鍵內容**:
  - CVSS 評分：10.0（滿分，極度嚴重）
  - 影響範圍：React Server Components、Next.js 及相關框架
  - 漏洞類型：預認證遠程代碼執行（Pre-authentication RCE）
  - 攻擊方式：單個惡意 HTTP 請求即可執行任意代碼
  - 首次發現攻擊：2025年12月5日
  - 風險因素：
    - 默認配置即存在漏洞
    - 公開的 PoC 漏洞利用代碼可用性高（近 100% 可靠性）
    - 無需用戶認證即可利用
    - 單個 HTTP 請求即可完成攻擊
  - 根本原因：React Server Components 未能驗證傳入的 payload，導致原型污染和遠程代碼執行
  - 包含 CVE-2025-66478（已合併到此漏洞中）
