# 已驗證的新參考資料

## 1. AI Toolkit for VS Code — February 2026 Update
- **來源**：Microsoft Developer Community Blog
- **日期**：2026-02-13
- **權重**：最高（Microsoft 官方）
- **URL**：https://techcommunity.microsoft.com/blog/azuredevcommunityblog/%F0%9F%9A%80-ai-toolkit-for-vs-code-%E2%80%94-february-2026-update/4493673
- **重點**：
  - 發布 v0.30.0 版本，引入 Tool Catalog、Agent Inspector、Evaluation as Tests 三大功能
  - Tool Catalog 提供集中式工具管理，整合 Foundry 目錄與 MCP 伺服器
  - Agent Inspector 支援 F5 一鍵除錯，提供斷點、變數檢查與即時視覺化
  - Evaluation as Tests 將品質檢查整合到 pytest 語法與 VS Code Test Explorer
  - Agent Builder 重新設計介面，支援 Foundry prompt agents
  - 新增 OpenAI Response API 支援，包括 gpt-5.2-codex
- **對程式設計師的影響**：直接影響 AI 代理開發流程，提供生產級別的除錯與測試工具


## 2. The Best AI Models for Coding: Accuracy, Integration, and Developer Fit
- **來源**：JetBrains AI Blog
- **日期**：2026-02-11
- **權重**：最高（JetBrains 官方）
- **URL**：https://blog.jetbrains.com/ai/2026/02/the-best-ai-models-for-coding-accuracy-integration-and-developer-fit/
- **重點**：
  - 基於 JetBrains Developer Ecosystem Report 2025 的深度分析
  - 比較 GPT-5/5.1、Claude 3.7 Sonnet、Gemini 2.5 Pro、DeepSeek R1 等主流編碼模型
  - 評估標準包括：程式碼品質、IDE 整合、價格授權、數據隱私、本地部署、客製化能力
  - 2025 年有高達 85% 的開發者定期使用 AI 編碼工具
  - 開發者最關注的是程式碼品質與 IDE 整合度
  - 企業團隊則優先考慮合規性、治理與長期可擴展性
- **對程式設計師的影響**：提供權威的 AI 編碼工具選擇指南，幫助開發者與團隊做出明智決策
