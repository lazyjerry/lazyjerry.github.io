## 6. AWS 與 Google Cloud 合作推出多雲網路解決方案
- **來源**：Google Cloud Blog（官方）
- **發布日期**：2025年12月1日
- **標題**：AWS and Google Cloud collaborate to simplify multicloud networking
- **網址**：https://cloud.google.com/blog/products/networking/aws-and-google-cloud-collaborate-on-multicloud-networking
- **共同發布者**：Rob Enns (VP/GM of Cloud Networking, Google Cloud) 與 Robert Kennedy (VP of Network Services, AWS)

### 重點內容

#### 合作背景
- 隨著企業越來越多採用多雲架構，雲端服務供應商之間的互通性需求前所未有
- 過去連接這些環境一直是挑戰，客戶被迫採用複雜的「自己動手」方式管理全球多層網路
- 傳統方式需要數週甚至數月的時間來設置物理連接和設備

#### 技術解決方案
- 聯合開發的多雲網路解決方案使用 **AWS Interconnect - multicloud** 和 **Google Cloud's Cross-Cloud Interconnect**
- 引入新的開放規範（open specification）實現網路互通性
- 客戶可在 Google Cloud 和 AWS 之間建立私有、高速連接，具有高度自動化和速度
- 從物理基礎設施管理轉向託管的雲原生體驗

#### 關鍵優勢
1. **快速部署**：客戶不再需要等待數週的電路配置，現在可以按需配置專用頻寬，並在幾分鐘內透過首選雲端控制台或 API 建立連接
2. **高可靠性**：利用四重冗餘（quad-redundancy），跨越物理冗餘的互連設施和路由器
3. **安全性**：使用 MACsec 加密技術保護 Google Cloud 和 AWS 邊緣路由器之間的連接
4. **持續監控**：兩家供應商持續監控以主動檢測和解決問題

#### 開放標準
- 為此產品開發的 **API 規範是開放的**，供其他供應商和合作夥伴採用
- 目標是為所有人簡化全球連接
- Azure 預計將在 2026 年加入

#### 客戶見證
- **Salesforce**：整合 Salesforce Data 360 需要強大的私有連接，AWS Interconnect - multicloud 讓他們能夠像部署內部 AWS 資源一樣輕鬆建立與 Google Cloud 的關鍵橋樑

#### 產業意義
- 這不僅是多雲解決方案，更是邁向更開放雲端環境的一步
- 抽象化物理連接、網路定址和路由策略的複雜性
- 簡化全球連接，提升營運效能


## 7. OpenAI 共同創立 Agentic AI Foundation
- **來源**：OpenAI（官方）
- **發布日期**：2025年12月9日
- **標題**：OpenAI co-founds the Agentic AI Foundation under the Linux Foundation
- **網址**：https://openai.com/index/agentic-ai-foundation/

### 重點內容

#### 基金會成立
- OpenAI 與 Anthropic、Block 共同在 Linux Foundation 下創立 **Agentic AI Foundation (AAIF)**
- 獲得 Google、Microsoft、AWS、Bloomberg、Cloudflare 支持
- 目標：為 agentic AI 系統提供中立管理，促進開放、可互通的基礎設施

#### AGENTS.md 標準
- OpenAI 貢獻 **AGENTS.md** 格式給基金會
- 這是一個簡單、開放的格式，為 AI 代理提供專案特定的指令和上下文
- 設計為與 README.md 並存的輕量級 Markdown 文件
- 自 2025 年 8 月發布以來，已被超過 60,000 個開源專案採用
- 支援的框架包括：Amp、Codex、Cursor、Devin、Factory、Gemini CLI、Github Copilot、Jules、VS Code 等

#### 基金會貢獻專案
1. **OpenAI**：AGENTS.md
2. **Anthropic**：Model Context Protocol (MCP)
3. **Block**：goose

#### OpenAI 的開源貢獻
- Agents SDK
- Apps SDK
- Agentic Commerce Protocol
- gpt-oss models
- Codex CLI（已協助合併超過 200 萬個 GitHub 公開 pull requests）
- ChatGPT 中的 connectors 和 apps

#### 產業意義
- 防止 AI 代理生態系統碎片化
- 建立共同標準，使代理更安全、更易於構建、跨工具和平台更具可攜性
- Linux Foundation 提供中立治理、社群建設和長期可持續性的專業經驗
- 開放標準確保沒有單一公司控制基礎設施的方向

#### 實際應用
- 從編碼助手到工作流程自動化和客戶服務代理
- 2025 年這些系統已從原型轉變為處理商業和消費者環境中實際工作的工具
- 提供專案特定指導（如編碼慣例、建置步驟、測試要求）

## 10. Docker Hardened Images 免費開源
- **來源**：Docker Blog（官方）
- **發布日期**：2025年12月17日
- **標題**：A Safer Container Ecosystem with Docker: Free Docker Hardened Images
- **網址**：https://www.docker.com/blog/docker-hardened-images-for-every-developer/
- **作者**：Christian Dupuis 和 Michael Donovan

### 重點內容

#### 產業背景
- Docker Hub 每月超過 200 億次拉取
- 近 90% 的組織在軟體交付工作流程中依賴容器
- 2025 年供應鏈攻擊造成超過 600 億美元損失，較 2021 年增加三倍

#### 重大宣布
- Docker Hardened Images (DHI) 現已免費開源，採用 **Apache 2.0 授權**
- 向全球 2,600 萬以上的容器生態系統開發者免費提供
- 自 2025 年 5 月推出以來，已強化超過 1,000 個映像檔和 Helm charts

#### 技術特點
1. **最小化攻擊面**
   - 使用 distroless runtime 縮小攻擊面
   - 映像檔體積減少高達 95%
   - 保證接近零 CVE（在 DHI Enterprise 中）

2. **完全透明**
   - 每個映像檔包含完整且可驗證的 SBOM
   - 每個建置提供 SLSA Build Level 3 來源證明
   - 使用透明的公開 CVE 資料評估每個漏洞
   - 不會隱藏未修復的漏洞

3. **相容性**
   - 基於 Alpine 和 Debian，團隊可以最小摩擦採用
   - Docker AI 助手可掃描現有容器並推薦或應用等效的強化映像檔

- 支援受監管產業（如 FIPS、FedRAMP）

## 11. NVIDIA 和 AMD 評估 Intel 14A 製程，蘋果、博通測試 EMIB 先進封裝

**來源**: TechNews 科技新報
**日期**: 2025-12-19
**網址**: https://technews.tw/2025/12/19/nvidia-and-amd-evaluate-intels-14a-process-technology-apple-and-broadcom-test-emib-advanced-packaging/

**關鍵內容**:
- **NVIDIA 和 AMD** 正積極評估 **Intel 14A 製程節點**，作為台積電產能吃緊下的替代方案
- **蘋果（Apple）** 和 **博通（Broadcom）** 考慮採用 Intel 的 **EMIB 先進封裝技術**，用於客製化伺服器晶片
- 蘋果已完成 **Intel 18A-P 製程設計套件（PDK）0.9.1 版** 的初步測試，等待 2026 年 Q1 發布的 PDK 1.0/1.1 版本
- 博通協助開發的蘋果客製化伺服器晶片「**Baltra**」，原計劃採用台積電 N3 製程，但因 **CoWoS 先進封裝產能吃緊**，轉向考慮 Intel EMIB 封裝
- 該客製化 AI 伺服器晶片預計 **2028 年量產**，採用 Intel 製程的 M 系列晶片最早可能在 **2027 年問世**
- **Intel 14A 製程** 標榜在每瓦性能與晶體管密度上具有顯著優勢，並能整合 EMIB 與 Foveros 等先進封裝技術
- 此舉反映晶片設計大廠在 AI 浪潮下，試圖擺脫單一供應鏈的依賴風險

**影響分析**:
這是半導體產業供應鏈重組的重要信號，顯示 Intel 代工事業（IFS）正在獲得頂級客戶的認可。對程式設計師而言，這意味著未來可能會有更多元化的硬體平台選擇，特別是在 AI 和高效能運算領域。
