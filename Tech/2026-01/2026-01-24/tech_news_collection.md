### GitHub Copilot SDK (Jan 22, 2026)
- **發布日期**: 2026年1月22日
- **來源**: GitHub Blog
- **URL**: https://github.blog/news-insights/company-news/build-an-agent-into-any-app-with-the-github-copilot-sdk/
- **作者**: Mario Rodriguez (Chief Product Officer)

**核心內容**:
- GitHub Copilot SDK 進入技術預覽階段（technical preview）
- 允許開發者將 GitHub Copilot CLI 的 agentic core 嵌入任何應用程式
- 提供程式化存取權限，使用與 GitHub Copilot CLI 相同的生產測試執行迴圈
- 支援多個 AI 模型、自訂工具定義、MCP 伺服器整合、GitHub 身份驗證和即時串流

**Copilot CLI 新功能**:
- 持久記憶、無限會話和智慧壓縮
- explore、plan 和 review 工作流程，可選擇每個步驟使用的模型
- 自訂代理、代理技能、完整 MCP 支援和異步任務委派

**SDK 應用案例**:
- YouTube 章節生成器
- 代理的自訂 GUI
- 語音到命令工作流程
- 與 AI 競爭的遊戲
- 摘要工具

**技術特點**:
- 作為執行平台，重用 Copilot CLI 背後的 agentic loop
- GitHub 處理身份驗證、模型管理、MCP 伺服器、自訂代理和聊天會話加串流
- 開發者可控制在這些構建塊之上構建的內容


### Anthropic Claude for Healthcare and Life Sciences (Jan 11, 2026)
- **發布日期**: 2026年1月11日
- **來源**: Anthropic News
- **URL**: https://www.anthropic.com/news/healthcare-life-sciences

**核心內容**:
Anthropic 推出 Claude for Healthcare 和擴展 Claude for Life Sciences 功能

**Claude for Healthcare 新功能**:
1. **Connectors（連接器）**:
   - CMS Coverage Database（Medicare & Medicaid 覆蓋資料庫）
   - ICD-10（國際疾病分類第10版）
   - National Provider Identifier Registry（國家醫療提供者識別註冊表）
   - PubMed（超過3500萬篇生物醫學文獻）

2. **Agent Skills**:
   - FHIR 開發技能（醫療系統數據交換標準）
   - Prior authorization review（事前授權審查範本）

3. **個人健康數據整合**（Beta）:
   - HealthEx 和 Function 連接器
   - Apple Health 和 Android Health Connect 整合
   - 用戶可控制分享資訊，隨時斷開連接
   - 不使用用戶健康數據訓練模型

**醫療應用場景**:
- 加速事前授權請求審查
- 支援索賠申訴
- 協調護理和分流患者訊息
- 支援醫療新創開發新產品

**Claude for Life Sciences 擴展**:
1. **新增 Connectors**:

### InfoWorld - 6 AI Breakthroughs That Will Define 2026 (Jan 9, 2026)

- **來源**: InfoWorld
- **作者**: Dwarak Rajagopal (VP of AI Engineering at Snowflake)
- **URL**: https://www.infoworld.com/article/4108092/6-ai-breakthroughs-that-will-define-2026.html

**6 大 AI 突破**:

1. **Open-source models will break the hold of AI giants（開源模型將打破 AI 巨頭壟斷）**
   - 突破發生在後訓練階段（post-training phase）
   - 開源模型可針對特定應用客製化和微調
   - 新創公司和研究人員可在共享開放基礎上創建強大的客製化 AI 解決方案

2. **Improvements in context windows and memory will drive agentic innovation（上下文視窗和記憶改進將推動代理創新）**
   - 上下文視窗和改進的記憶將成為 agentic AI 創新的主要驅動力
   - 賦予代理持久記憶，從過去行動中學習
   - 使代理能夠自主處理複雜、長期目標

3. **Self-verification will start to replace human intervention（自我驗證將開始取代人工干預）**
   - 配備內部反饋迴圈，自主驗證工作準確性並糾正錯誤
   - 「自動判斷」代理將使複雜的多跳工作流程既可靠又可擴展
   - 從概念驗證轉變為可行的企業解決方案

4. **English will become the hottest new programming language（英語將成為最熱門的新程式語言）**
   - AI 生成和執行程式碼的能力提供關鍵橋樑
   - 主要技能不再是特定語法，而是能夠清楚表達目標
   - 軟體開發民主化，創作者數量將增加十倍

5. **The AI arms race will shift from bigger models to smarter ones（AI 軍備競賽將從更大模型轉向更智慧模型）**
   - 2025 年觸及 Chinchilla formula 等擴展定律的極限
   - 高品質預訓練數據耗盡
   - 創新快速轉向後訓練技術（post-training techniques）
   - 焦點從模型規模轉向精煉和專業化

6. **Agent interoperability will unlock the next wave of AI productivity（代理互操作性將解鎖下一波 AI 生產力）**
   - 開發開放標準和協議，允許不同 AI 代理相互通信
   - 「代理經濟」將允許不同平台的代理自主發現、協商和交換服務
   - 解鎖複合效率，自動化目前不可能的複雜多平台工作流程

**核心觀點**:
- 最重要的進展不是建構更大的模型，而是讓 AI 系統更智慧、更協作、更可靠
- 產業正在解決阻止 AI 在生產環境中可靠運作的實際問題
- 「更大就是更好」的時代已讓位給「更智慧才是必要」的時代
